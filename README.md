# Linux环境下基础运维流程实践（Nginx服务监控与故障处理）

---

## 项目简介

本项目用于模拟基础运维工作流程，通过搭建简单的Nginx服务环境，完成从服务状态确认、日志分析到故障排查与恢复的完整流程实践。

该项目重点在于理解运维流程，而非系统开发或复杂架构设计。

---

## 项目目标

- 掌握基础服务状态确认方法（systemctl）
- 理解日志在运维中的作用（access.log / error.log）
- 熟悉基础故障排查流程
- 能够完成简单的服务恢复操作

---

## 环境

- 操作系统：Ubuntu（WSL）
- Web服务：Nginx
- 工具：
  - systemctl
  - curl
  - tail
  - grep

---

## 运维流程实践

---

### 场景1：服务正常运行

#### 操作

```bash
systemctl status nginx
curl localhost
```

#### 现象

- Nginx服务处于 active (running)
- 页面可以正常访问
- 访问日志（access.log）中记录请求信息

#### 结论

服务运行正常，可以通过服务状态和日志进行确认。

---

### 场景2：服务异常（故障模拟）

#### 操作（模拟故障）

```bash
sudo systemctl stop nginx
```

#### 现象

```bash
curl localhost
```

- 无法访问（connection refused）

---

### 场景3：故障排查过程

#### 1. 服务状态确认

```bash
systemctl status nginx
```

结果：

- nginx 为 inactive (dead)

---

#### 2. 日志确认

```bash
tail -n 20 /var/log/nginx/error.log
```

说明：

- 日志中未发现明显错误信息，因此结合服务状态判断问题原因

---

#### 3. 问题判断

- 服务未运行（nginx停止）
- 导致无法提供Web访问

---

### 场景4：故障处理与恢复

#### 执行恢复

```bash
sudo systemctl start nginx
```

---

#### 验证恢复

```bash
curl localhost
```

结果：

- 页面恢复正常访问

---

## 运维流程总结

本项目完成了基础运维流程实践：

问题发现 → 状态确认 → 日志分析 → 原因判断 → 执行恢复

本项目基于以下基础运维流程进行操作验证：

状态确认 → 日志确认 → 问题判断 → 执行恢复 → 结果验证
