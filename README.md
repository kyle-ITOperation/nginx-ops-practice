# Nginx Operations Practice

## 项目概述
本项目用于模拟Linux环境下Web服务运维工作，
以Nginx服务为核心，围绕服务状态确认、日志分析、
访问验证及异常处理流程进行实践。

项目时间：2026.03（集中实践）

---

## 运维流程

状态确认 → 访问验证 → 日志检查 → 问题判断 → 基础处理 → 结果验证

---

## 日常运维作业内容

- 服务状态检查（systemctl）
- 访问验证（curl）
- 日志检查（access.log / error.log）
- 服务异常基础处理（重启等）
- 处理结果验证

## 故障处理案例

### 案例1：Nginx服务未运行

【问题现象】
- curl访问失败  

【处理步骤】
1. systemctl检查服务状态  
2. 确认服务未运行  
3. 执行服务重启  

【处理结果】
- 服务恢复正常  

---

### 案例2：访问异常（日志报错）

【问题现象】
- curl访问异常  

【处理步骤】
1. 查看error.log  
2. 确认错误信息  
3. 进行基础判断  

【处理结果】
- 记录问题并处理
