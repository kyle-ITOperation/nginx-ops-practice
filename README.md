# Nginx Operations Practice

## 项目概述

本项目围绕Linux环境下Web服务（Nginx）的日常运维作业进行整理，涵盖服务状态确认、访问验证、日志检查及异常处理流程。

结合实际运维场景，对服务异常时的排查步骤进行记录，用于强化在运维环境中的基础操作及问题判断能力。

项目时间：2026.03（集中实践）

---

## 运维流程

状态确认 → 访问验证 → 日志检查 → 问题判断 → 基础处理 → 结果验证

---

## 日常运维作业

- 使用systemctl确认服务运行状态  
- 使用curl进行访问验证（确认服务响应）  
- 查看access.log确认访问情况  
- 查看error.log确认异常信息  
- 在服务异常时进行基础检查及处理
- 在服务访问异常或响应异常时进行排查

---

## 故障处理案例

### 案例1：服务无法访问

现象：
- curl访问失败  

处理过程：
1. 使用systemctl确认服务状态  
2. 查看error.log确认异常信息  
3. 执行服务重启  
4. 再次进行访问验证  

结果：
- 服务恢复正常

---

### 案例2：访问异常（日志报错）

【问题现象】
- curl访问异常  

【处理步骤】
1. 查看error.log  
2. 确认错误信息  
3. 进行基础判断  

【处理结果】
- 记录问题并处理
